# Task Service (MIS Module)

Данный проект представляет собой API модуля трекера задач для медицинской информационной системы (МИС). Основная задача обновления — реализация функционала периодических задач для медицинского персонала.

## Реализованный функционал

1. **Типы периодичности (согласно ТЗ):**
   - **Ежедневные:** выполнение задачи каждые N дней.
   - **Ежемесячные:** задачи на конкретное число месяца (с корректной обработкой месяцев разной длины).
   - **Конкретные даты:** создание задач строго по переданному списку дат.
   - **Четные/нечетные:** задачи создаются по календарной четности дня месяца.
2. **Фильтрация:** Возможность получения плана задач на конкретный день: `GET /api/v1/tasks?date=YYYY-MM-DD`.
3. **Документация:** Полная поддержка OpenAPI 3.0. Все новые поля и параметры фильтрации доступны в Swagger UI.

## Архитектурные решения и обоснование

### Стратегия упреждающей генерации экземпляров (Pre-generation)
При проектировании была выбрана стратегия создания физических записей в БД на 30 дней вперед вместо динамического вычисления периодичности при каждом запросе.

*   **Гибкость для персонала:** Это позволяет врачу редактировать описание или менять статус конкретного экземпляра задачи (например, добавить примечание к обходу в конкретный четверг), не затрагивая остальную серию.
*   **Производительность:** Выборка "задач на сегодня" выполняется за константное время обычным SQL-запросом.
*   **Прозрачность:** План работ на ближайший месяц сразу виден в базе данных и API.

### Технические оптимизации
- **Batch Insertion:** Использование `pgx.CopyFrom` для массового создания задач. Это позволяет вставить всю серию одним пакетом, минимизируя количество сетевых запросов к PostgreSQL.
- **Data Consistency:** Все дочерние задачи связаны с оригинальным шаблоном через поле `parent_id`. Это закладывает фундамент для будущих функций каскадного удаления или обновления всей серии.
- **Безопасность типов:** Логика защищена от nil-pointer десериализации, а в БД добавлены индексы на поля `scheduled_at` и `parent_id`.
